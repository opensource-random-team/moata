# 🚖 Moata Project - 택시 동승 매칭 프로젝트

> **택시비 부담을 줄이는 스마트한 동승 매칭 서비스**

---

## 📌 프로젝트 개요

**moata**는 택시비의 부담으로 인해 택시 이용을 주저하는 사람들을 위한 **택시 동승 및 더치페이 서비스**입니다.  
위치 기반 시스템을 통해 **같은 목적지를 향하는 사용자들을 자동으로 매칭**시켜주며,  
이를 통해 **비용 부담 절감**, **택시 이용 효율 증가**, **환경 보호 효과**를 기대할 수 있습니다.

---

## ⚙️ 주요 기능

### 🗺️ 1. 위치 기반 시스템
- 주변에 같은 서비스를 사용하는 사람들의 위치를 탐색
- 동일한 목적지를 가진 사용자 간 자동 매칭

### 💬 2. 커뮤니티 기능
- 위치 기반 매칭이 어려울 경우 직접 글을 게시하여 동승자 모집
- 원하는 시간, 날짜, 목적지에 맞춰 동승자 모집 글 작성 가능
- 댓글 및 피드백 기능 제공

### 💭 3. 실시간 채팅 기능
- 매칭된 사용자 간 **실시간 소통** 가능  
- **만남 장소, 시간, 요금 분할 방식** 등을 빠르게 협의 가능  
- WebSocket 기반 실시간 채팅 기능 지원

## 🌱 프로젝트 수행 후 느낀 점

### 👤 문서연 (Backend)

- WebSocket을 이용해 실시간 채팅을 구현하면서 메시지를 바로바로 주고받기 위해 필요한 연결 관리 구조와 비동기 처리 방식을 어느 정도 이해할 수 있게 되었다. 로그인에서의 Spring Security를 적용하면서는 권한 체크, 인증 등 내부에서 처리해주는 것들이 있어서 조금 힘들었다. 또, 매칭 기능에서는 사용자 위치를 제대로 찾아주지 못해 로직을 바꿔보는 등 여러가지를 시도하였지만, 결국은 PC에서만 위치를 잘 찾아주지 못하고 핸드폰에선 정확도가 높게 위치를 찾아주는 것을 확인하였다. 이번 프로젝트를 통해 단순히 기능을 만들어내는 것이 아니라 서비스가 안정적으로 동작할 수 있게 구조를 설계하는 방법을 배우게 된 것 같다.

### 👤 윤은혜 (Frontend)

- 이번 프로젝트를 통해 단순한 이론 학습을 넘어, 아이디어를 실제 서비스로 구현하는 과정에서 개발자로서의 역량을 크게 향상시키는 계기가 되었다. 특히, 협업과 버전 관리에 필수적인 Git과 GitHub 활용법을 실제 개발 환경에서 몸소 익힐 수 있었다. 책이나 강의를 통한 학습이 아닌, 충돌 해결과 브랜치 관리 등의 실전 경험을 통해 지식이 오랫동안 기억에 남았으며, 실행이 이론에 앞선다는 개발의 핵심 가치를 깨닫게 되었다. 사용자의 니즈를 분석하고, 이를 웹페이지의 실제 기능으로 구현하는 일련의 과정에서 큰 재미와 성취감을 느꼈다. 특히, 외부 지도 API를 연동하여 사용자의 실시간 위치를 파악하고 이를 화면 상에서 표시하여 즉각적인 결과를 확인하는 과정이 가장 인상깊었고 흥미로웠다.

### 👤 최희원 (Backend)

- 이번 프로젝트를 통해 웹사이트가 어떤 동작으로 작동하는지 자세하게 알 수 있었고, 반복의 중요성을 느꼈다. 여러 화면과 기능이 서로 연결되면서 예상치 못한 오류도 많았지만, 해결하는 과정에서 JPA, 컨트롤러, gradle, 백엔드와 프론트의 연결, 비밀번호 보안, DB 구조, 서버 대한 이해가 크게 높아졌다. 깃허브도 많이 이용해보면서 PR, merge, pull 등의 기능을 익힐 수 있었고 약간의 conflict도 겪으면서, 팀 협업의 방법과 중요도에 대해 알 수 있었다.

## 🔮 향후 발전 방향

- 매칭 고도화: AI 기반의 '수요 예측 및 최적 경로 분석' 알고리즘 도입. (ex: 실시간 교통량, 날씨 반영)	매칭 성공률 및 동승 효율 극대화, 요금 절감 효과 증대
- 안전 및 신뢰:	'신뢰 지수(Trust Score)' 시스템 도입 및 '안심 귀가 동행 공유' 기능 의무화.	이용자 간의 신뢰 확보 및 서비스의 안정성/경쟁력 강화
- B2B 기업 통근 솔루션: 여러 기업의 직장인 통근 수요를 통합하여 최적화된 **'맞춤형 셔틀/카풀 매칭 서비스'**를 제공, 기업의 운영 비용 절감 및 직원 복지 증진에 기여
- 공공 교통 연계 (DRT): 대중교통이 취약한 지역이나 시간대의 '수요 응답형 교통(DRT, Demand Responsive Transit)' 시스템에 동승 매칭 기술을 적용하여, 공공 교통의 효율성과 접근성을 혁신적으로 개선
